# 미션 - 숫자 야구 과제 수행

## 유저 시나리오

1.  사용자가 어플을 실행한다.
2.  내부적으로 정답(3개의 겹치지 않는 무작위 숫자)을 생성한다.
3.  사용자가 입력을 한다.
4.  사용자가 입력한 사항과 정답을 비교한다.
5.  비교의 결과에 따라 출력을 한다.

- 하나도 없는 경우 -> "낫싱"
- 하나라도 있는 경우 -> "볼" || "스트라이크"
- 모두 다 맞는 경우 -> "3스트라이크"

6.  정답이 아닌 경우 3~5의 사항을 반복한다.
7.  정답인 경우 게임을 종료하고 재시작 여부를 묻는다.

정리하자면 **start -> process(validation, checkAnswer) -> restart**의 순서대로 작동하는 로직을 구현해야한다.
