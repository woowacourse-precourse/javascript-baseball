# commit message convention

- subject types

  - feat
  - fix
  - docs
  - style
  - refactor
  - test
  - chore : package.json의 변경이나 dotenv의 요소 변경 등, 모듈의 변경

- body rule (영어를 사용할 경우)

  - 과거형이 아닌 명령어, 현재형으로
  - 첫번째 문자는 소문자로
  - 마지막은 마침표가 없도록

- 예 ) feat: add lostMap component
